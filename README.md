## Business Problem

Develop a system to generate personalized care plans for patients with chronic diseases (diabetes, hypertension, CKD) using their medical history, lifestyle data, and genetic risk proxies.

**Day 1 objectives:**
1. **Patient Segmentation** — cluster patients into clinically interpretable risk/behaviour groups (K-Means + Hierarchical clustering).
2. **Complication Risk Prediction** — classify individual patients as high-risk or low-risk using Logistic Regression, Random Forest, and Gradient Boosting.

Results are stored with patient IDs, cluster labels, and risk scores so subsequent phases can consume them directly.

## Synthetic Dataset — Generation Logic

**File:** `data/raw/patients_raw.csv` | **Rows:** 6,000 | **Columns:** 23

The dataset is **entirely synthetic** and was generated by `src/generate_data.py` using `numpy` random distributions with clinically plausible parameters. **No real patient data is used.**

### Feature generation summary

| Column | Distribution / Logic |
|---|---|
| age | `Normal(55, 15)` clipped to [18, 85] |
| sex | Bernoulli, 52% F |
| bmi | `LogNormal(ln(27), 0.2)` clipped to [16, 45] |
| smoking_status | Multinomial: 50% never, 30% former, 20% current |
| alcohol_use | Multinomial: 45% none, 40% moderate, 15% heavy |
| hba1c | `Normal(5.0 + 2.5×diabetes_flag, 1.2)` — correlated with diabetes |
| fasting_glucose | `Normal(90 + 70×diabetes_flag, 30)` — correlated with diabetes |
| systolic_bp | `Normal(115 + 25×hypertension_flag, 15)` |
| diastolic_bp | `Normal(75 + 10×hypertension_flag, 10)` |
| creatinine | `0.9 + 2.0×ckd_flag + Exponential(0.3)` — correlated with CKD |
| genetic_risk_score | `Beta(2, 5)` — right-skewed, most patients have low genetic risk |
| medication_adherence_pct | `Beta(5, 2) × 100` — most patients are moderately adherent |

### Target variable: `high_risk_complication` (binary, ~30% positive class)

Generated from a weighted logistic sigmoid:

```
log_odds = -5.0
    + 0.6 × (hba1c − 6.5)
    + 0.02 × (fasting_glucose − 100)
    + 0.03 × (systolic_bp − 120)
    + 0.08 × (bmi − 25)
    + 1.2 × creatinine
    + 2.0 × genetic_risk_score
    + 0.5 × hypertension_flag
    + 0.8 × diabetes_flag
    + 0.6 × ckd_flag
    + 0.3 × family_history_flag
    − 0.02 × medication_adherence_pct
    − 0.2 × diet_score
    + activity_penalty
    + Gaussian noise (σ=0.5)

high_risk = 1  if  sigmoid(log_odds) > 0.45
```

### Injected messiness

- **5–8% missing values** in: `hba1c`, `fasting_glucose`, `creatinine`, `medication_adherence_pct`, `diet_score`
- **~1% outliers** in `hba1c`, `fasting_glucose`, `systolic_bp` (values above clinical maximums)
- **~70/30 class imbalance** in `high_risk_complication`

## Preprocessing Decisions

See `src/data_prep.py` for full documentation. Key decisions:

| Step | Method | Rationale |
|---|---|---|
| Lab value imputation | Median | Robust to the outliers we injected; median is clinically more conservative |
| Diet / adherence imputation | Mean | Normally distributed, mean imputation appropriate |
| Categorical imputation | Mode | Preserves most frequent clinical category |
| Outlier handling | IQR winsorizing (×1.5) | Caps extreme lab values rather than dropping rows |
| Encoding | One-Hot for nominals, Ordinal for ordered | Prevents false ordinality in nominals |
| Scaling | StandardScaler | Required for LR, SVM, K-Means distance metrics |
| Train/test split | Stratified 80/20 | Preserves ~30% positive class in both sets |

## Feature Engineering

See `src/features.py`. Three derived features:

| Feature | Formula | Clinical Rationale |
|---|---|---|
| `bmi_category` | WHO thresholds (<18.5 / 18.5–24.9 / 25–29.9 / ≥30) | Obesity is a direct chronic disease risk factor |
| `comorbidity_count` | `hypertension_flag + diabetes_flag + ckd_flag` | Cumulative burden predicts complication risk |
| `risk_adjusted_adherence` | `medication_adherence_pct × (1 − genetic_risk_score)` | Adherence value is diminished by high genetic risk |

## Reproducing Results

All random seeds are fixed at `RANDOM_SEED = 42` in every script. Running scripts in this order will produce identical results:

```bash
python src/generate_data.py   # Step 1: generate data
python src/data_prep.py       # Step 2: preprocess
python src/train.py           # Step 3: train models
python src/evaluate.py        # Step 4: evaluate & report
```
